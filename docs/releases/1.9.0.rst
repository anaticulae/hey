RP 1.9.0
=========

At the end of `hey release 1.9.x` we want to fully support detecting
footer and header. In the following doc footer contains header and
footer but it is only written footer. This contains the following steps:

Footer
------

FixedFooter
~~~~~~~~~~~
Footer and Header are fixed, that means for every page,
there is a header and or footer with the same distance. It is possible
that some page have no header or footer.

MovingFooter
~~~~~~~~~~~~
The moving header means that the footer is not fixed. As a
result of different foot notes it is possible that the footers have
different sizes.

PagesFooter
~~~~~~~~~~~
A PageFooter is defined by page number in header or
footer, but there is no other content in footer. It is also possible
that there is a fixed headline in the component.

StrategyDecider
~~~~~~~~~~~~~~~
Decide which result fits best.
